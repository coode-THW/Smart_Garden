Phase 1：核心识别与基础功能（第 1-40 天）

目标：用户能拍照识花、查看养护指南、管理花园。零注册可用。

第 1 单元：环境搭建与技术验证（第 1-10 天）

天数A · AI 工程师B · 前端工程师C · 全栈工程师
Day 1-2搭建 Python 训练环境；导出 ONNX 模型搭建 RN 开发环境；npx react-native init阅读文档；搭建 Git 仓库；准备知识库 JSON 模板
Day 3-4验证 ONNX 模型 Python 端推理（10 张测试图）安装依赖：onnxruntime-react-native、paper、zustand设计 SQLite 表结构；编写建表 SQL
Day 5-6编写 ONNX 推理最小示例；真机跑通 InferenceSession搭建导航框架；创建首页+识别页+花园页实现 SQLite 初始化模块 + CRUD 基础函数
Day 7-8编写图片预处理（resize→归一化→Tensor）集成相机组件；拍照→保存本地路径实现 JSON 读取模块；创建首份月季.json
Day 9-10🔗 集成测试：A+B 联调拍照→YOLO推理→返回结果验证 SQLite 读写+JSON 读取；首个单元测试
📦 交付物：ONNX 模型真机推理成功 · RN 项目骨架搭建完成 · 3 张数据库表就绪 · 首份养护 JSON 验证通过 · A+B 联调跑通

第 2 单元：识别核心闭环（第 11-20 天）

天数A · AI 工程师B · 前端工程师C · 全栈工程师
Day 11-12实现三级置信度判断逻辑识别结果页 UI——花名、置信度、学名、科属编写 10 份养护知识 JSON
Day 13-14实现 LlmService——Base64、Prompt、API结果页底部按钮：一键提醒+纠错编写 10 份养护知识 JSON
Day 15-16LLM 响应解析+模型路由（主备切换）相册选择识别；UI 状态提示编写 10 份养护知识 JSON
Day 17-18编写编排器——串联预处理→YOLO→决策→LLM启动预加载策略——加载画面+按钮状态编写剩余 20 份 JSON；总量达 50+ 品种
Day 19-20🔗 联调测试：完整识别流程（正常/低置信度/非花卉/离线四种场景）+ 50 张测试图验证
📦 交付物：完整识别闭环（YOLO+LLM）· 识别结果页完整 UI · 50+ 品种养护 JSON · 相册识别可用

第 3 单元：用户系统与花园管理（第 21-30 天）

天数A · AI 工程师B · 前端工程师C · 全栈工程师
Day 21-22优化图片预处理性能；推理结果缓存匿名用户初始化流程——自动生成 UUID实现 UserService——UUID 生成+可选绑定
Day 23-24非花卉过滤优化+测试集「我的花园」列表页——展示+删除实现 GardenService——添加/查询/删除
Day 25-26纠错反馈模块——记录到本地 SQLite添加花卉弹窗——自定义名称+位置+日期实现去重检查——SELECT flowerId 是否存在
Day 27-28性能测试——统计各机型推理耗时花园详情页——花卉信息+养护指南入口用户数据导出功能（SQLite→JSON）
Day 29-30🔗 全链路联调：拍照→识别→添加到花园→查看养护指南 + 回归测试
📦 交付物：匿名用户体系完成 · 花园管理完整（添加/去重/查看/删除）· 纠错反馈数据记录 · Phase 1 集成测试通过

第 4 单元：打磨与 Phase 1 收尾（第 31-40 天）

天数A · AI 工程师B · 前端工程师C · 全栈工程师
Day 31-32优化 LLM Prompt——提高养护建议质量首页 UI 优化——功能入口+最近识别知识库 JSON 内容校对——逐份检查准确性
Day 33-34模型量化 FP16→INT8 对比测试离线状态提示——网络检测+降级 UI错误码处理——所有错误 UI 提示文案
Day 35-36启动预加载完整逻辑——loading+超时处理欢迎页/启动页——品牌+模型加载进度日志系统——分级+本地文件滚动存储
Day 37-38Bug 修复+性能优化Bug 修复+UI 微调全功能回归测试；Phase 1 测试报告
Day 39-40📱 真机测试：Android 中端+低端 / iPhone 多机型兼容性验证
✅ Phase 1 交付物汇总：拍照识别花卉（≥50 种，准确率≥90%）· 三级置信度阈值 · 养护指南查询 · 免注册使用 · 花园管理 · 纠错反馈 · 启动预加载 · 离线可用 · 可演示版本

Phase 2：养护提醒与体验优化（第 41-60 天）

关键风险与应对

风险影响应对措施
ONNX Runtime 在低端机加载失败高第 1 单元尽早真机验证；准备 INT8 备选模型
模型识别准确率不达标高第 1 单元即验证；准备增量训练数据
LLM API 调用超频或额度不足中三级阈值过滤减少调用；充值备用金
Android 厂商后台杀进程→本地通知失效中引导用户开启自启动；Phase 3 远程推送兜底
应用商店审核不通过中提前阅读审核指南；准备隐私政策
三人时间协调困难低以接口协议为契约；各人独立开发，每 5 天对齐
