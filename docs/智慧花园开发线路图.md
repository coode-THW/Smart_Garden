# 智慧花园开发线路图

---

## 项目概况

| 属性 | 内容 |
|------|------|
| **项目名称** | 智慧花园（Smart Garden） |
| **技术架构** | React Native + ONNX Runtime 本地推理 + 大模型云端增强 |
| **目标平台** | Android / iOS |
| **团队规模** | 3 人（学生） |
| **时间单位** | 10 天 / 单元，共 8 个单元 |
| **总周期** | 80 天（约 2.5 个月） |
| **日均投入** | 约 3-4 小时 / 人 |
| **本文版本** | v1.0 |
| **创建日期** | 2026-06-12 |

---

## 人员角色分配

| 代号 | 角色 | 技术方向 | 主要职责 |
|------|------|----------|----------|
| **A** | AI 工程师 | Python / ONNX / LLM | YOLO 模型训练导出、ONNX 推理封装、LLM 调用、图片预处理 |
| **B** | 前端工程师 | React Native / UI | 页面开发、导航、状态管理、相机相册、交互体验 |
| **C** | 全栈工程师 | 数据 / 服务 / 测试 | SQLite 数据层、知识库 JSON、Nginx 服务、测试、文档 |

```
        A (AI/模型)          B (前端/UI)           C (数据/服务)
        ┌─────────┐         ┌─────────┐           ┌─────────┐
Phase 1 │ YOLO    │ ←──────→│ 识别页   │←────────→│ SQLite  │
        │ ONNX    │   接口   │ 相机    │   数据    │ JSON    │
        │ LLM     │   协议   │ 状态    │   模型    │ 测试    │
        └─────────┘         └─────────┘           └─────────┘
              │                   │                     │
Phase 2       │ 图片预处理优化     │ 提醒UI+通知          │ 提醒数据层
              │                   │                     │
Phase 3       │ 纠错数据格式       │ 推送SDK集成          │ Nginx+推送
```

## Phase 1：核心识别与基础功能（第 1-40 天）

> **目标**：用户能拍照识花、查看养护指南、管理花园。零注册可用。

| Day 5-6 | 编写 ONNX 推理最小示例，在模拟器/真机上跑通 `InferenceSession.create()` | 搭建导航框架（React Navigation）；创建首页 + 识别页 + 花园页三个空页面 | 实现 SQLite 初始化模块 `db.ts`；编写数据库 CRUD 基础函数 |

## 关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| ONNX Runtime 在低端机加载失败 | 高 | 第 1 单元尽早真机验证；准备 INT8 备选模型 |
| LLM API 调用超频或额度不足 | 中 | 三级阈值过滤减少调用；充值备用金 |
| Android 厂商后台杀进程导致本地通知失效 | 中 | 引导用户开启自启动权限；Phase 3 远程推送兜底 |
| 应用商店审核不通过 | 中 | 提前阅读审核指南；准备隐私政策 |
| 模型识别准确率不达标 | 高 | 第 1 单元即验证；准备增量训练数据 |
| 三人时间协调困难 | 低 | 以接口协议为契约；各人可独立开发，每 5 天对齐一次 |
