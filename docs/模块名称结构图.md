## 2. Flask Web 后端

> **定位**: Flask 是模型验证/演示工具，**不是**产品代码。产品是 React Native 移动端 APP。

```
YOLO_smart-garden/
```

## 附录 B：典型工作流与分支操作

### 场景 1：训练新模型

```
个人 training 分支                              main 分支
─────────────────                              ─────────
git checkout -b training/yolo-v2
python train.py                                 （不动）
  ↓ 产出 best.pt
python export_onnx.py
  ↓ 产出 models/yolov11n-flower.onnx
                                              git checkout main
git checkout main                              cp <新 onnx> models/
git merge training/yolo-v2 -- models/*.onnx    git add models/
git merge training/yolo-v2 -- export_onnx.py   git commit
git merge training/yolo-v2 -- model_utils.py
  ↑ 只合并 ONNX + 脚本，不含训练权重和数据集
```

### 场景 2：开发新功能

```
个人 feature 分支                               main 分支
─────────────────                              ─────────
git checkout -b feature/new-screen
# 在 SmartGarden/src/screens/ 下开发            （不动）
git add SmartGarden/src/**
git commit -m "feat: 新增 XXX 页面"
git push origin feature/new-screen
  ↓ 提 Pull Request
                                              git merge feature/new-screen
                                                ↑ 代码审查后合并
```

### 场景 3：更新养护知识库

```
个人分支 或 直接 main                           
─────────────────────────                      
# 编辑 SmartGarden/assets/care/月季.json        
git add SmartGarden/assets/care/**              
git commit -m "docs: 更新月季养护数据"           
git push origin main                            
  ↑ 可直接在 main 操作（数据量小、改动独立）     
```
